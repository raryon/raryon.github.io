Adding Structure by using HTML elements

An HTML web page needs to be organized and to do that there are tags created for structure. We have already seen that the html, head and body give the basic structure. Now we will look at additional ways to add further stucture. Think of a web page as being divided into different areas. We can use container tags to set the structure for each of those areas. A container is used to contain other elements, it is a way to help organize your elements.

The div container tag

The div html tag is used to create generic containers. The div element is a block element. Often there is one div tag that wraps all content, then there are other subcontainer div tags used to split the page up into different sections.

An id or class attribute is added to the div tag to give a name to that section so other statements can refer to it. For example CSS uses the id or class name to add special style features. There are div containers in this web page, look at the code to check it out.

The span container tag

Another container is created using the span tag. It is similar to the div except it is an inline element. Inline elements do not begin on new lines, but start from the next available position after the previous content on the same line. Inline content will also end at the end of the content.

HTML5 Semantic elements

HTML5 introduced additional structure elements know as semantic elements. These elements work similar to the div element but have specialized use. For example the nav element is used as a container for adding a navigation block. A header element is use to create a page heading. Also a footer element is used to add a block at the bottom of a web page.
